# Τεκμηρίωση και βοήθεια

<span
  class="summary">Αυτό το μάθημα παρουσιάζει τις κύριες πηγές τεκμηρίωσης για το λογισμικό και τα πακέτα που σχετίζονται με το LaTeX, καθώς και πώς να ζητήσετε βοήθεια όταν αντιμετωπίζετε προβλήματα.</span>

Υπάρχουν διάφοροι τρόποι πρόσβασης στην τεκμηρίωση ενός πακέτου ή μίας κλάσης (τύπου εγγράφου).

## `texdoc`

Αν έχετε εγκαταστήσει μία διανομή TeX (π.χ. TeX Live ή MiKTeX) και έχετε συμπεριλάβει
την τεκμηρίωση κατά την εγκατάσταση, μπορείτε να έχετε πρόσβαση στην τοπικά αποθηκευμένη
τεκμηρίωση χρησιμοποιώντας το εργαλείο γραμμής εντολών `texdoc`. Τρέχοντας:

`texdoc` < _pkg_ >

θα ανοίξει η τεκμηρίωση του πακέτου `<pkg>`. Το εργαλείο θα αναζητήσει στη
διαθέσιμη τεκμηρίωση και θα ανοίξει αυτό που θεωρεί ότι αντιστοιχεί καλύτερα στον
όρο της αναζήτησής σας. Μπορείτε να δείτε όλα τα σχετικά αποτελέσματα που εντοπίζει και να επιλέξετε αυτό που θέλετε
χρησιμοποιώντας:

`texdoc -l` < _pkg_ >

## texdoc.org

Πρόκειται για έναν [ιστότοπο](https://texdoc.org/) που λειτουργεί παρόμοια με το εργαλείο `texdoc`. Μπορείτε να αναζητήσετε
στην τεκμηρίωση που έχει διαθέσιμη, όπως θα κάνατε με το `texdoc -l`
και στη συνέχεια να επιλέξετε από τα αποτελέσματα.

## CTAN

Το [CTAN](https://www.ctan.org) είναι το Comprehensive TeX Archive Network. Τα περισσότερα πακέτα LaTeX δημοσιεύονται εκεί. Μπορείτε να χρησιμοποιήσετε το εργαλείο αναζήτησης του ιστότοπου για να βρείτε ένα πακέτο και να προσπελάσετε την τεκμηρίωσή του. Συνήθως, τα πακέτα αποθηκεύονται στο `ctan.org/pkg/<pkg-name>` και μπορείτε εκεί να δείτε το αρχείο README και την τεκμηρίωση των πακέτων που αποθηκεύονται στο CTAN.

## Βιβλία για το LaTeX

Υπάρχουν αρκετά βιβλία που μπορούν να σας βοηθήσουν να μάθετε περισσότερα για το LaTeX. Ως αρχάριος, θα κερδίσετε πολλά από έναν δομημένο οδηγό για αρχάριους, καθώς
μπορεί να παρέχει πολύ περισσότερες λεπτομέρειες από όσα έχουμε καλύψει εδώ. Μπορεί επίσης να σας ενδιαφέρει ένα βιβλίο αναφοράς με περισσότερες λεπτομέρειες και συστάσεις.

Η ομάδα του LaTeX έχει [μια λίστα βιβλίων](https://www.latex-project.org/help/books/)
που έχουν γραφεί κυρίως από μέλη της. Τα πιο αξιοσημείωτα είναι [το επίσημο βιβλίο του Lamport](https://www.informit.com/store/latex-a-document-preparation-system-9780201529838)
και το εκτενές
[LaTeX Companion 3rd edition](https://www.informit.com/store/latex-companion-parts-i-ii-3rd-edition-9780138166489).

Άλλα βιβλία που στοχεύουν στη μάθηση του LaTeX είναι τα εξής:

- [_Guide to LaTeX_](https://www.informit.com/store/guide-to-latex-9780132651714) των Helmut
  Kopka και Patrick Daly: διαθέσιμο ως e-book
- [_LaTeX for Complete Novices_](https://www.dickimaw-books.com/latex/novices/) της
  Nicola Talbot: διαθέσιμο ως δωρεάν e-book ή σε οικονομική εκτύπωση
- [_Using LaTeX to write a PhD
  thesis_](https://www.dickimaw-books.com/latex/thesis/) της
  Nicola Talbot: διαθέσιμο ως δωρεάν e-book ή σε οικονομική εκτύπωση
- [_LaTeX Beginner's Guide_](https://www.packtpub.com/gb/hardware-and-creative/latex-beginners-guide)
  του Stefan Kottwitz: διαθέσιμο ως e-book και σε έντυπη μορφή
- [_LaTeX and Friends_](https://www.springer.com/gp/book/9783642238154) του
  Marc van Dongen: διαθέσιμο ως e-book και σε έντυπη μορφή

## Λήψη βοήθειας

Υπάρχουν διάφορα διαδικτυακά φόρουμ για να κάνετε ερωτήσεις σχετικά με το LaTeX. Ίσως το πιο δημοφιλές σήμερα είναι το [TeX - LaTeX StackExchange](https://tex.stackexchange.com). Όταν κάνετε μια ερώτηση, είναι καλύτερο να έχετε προετοιμάσει ένα σαφές παράδειγμα: αυτό που είναι γνωστό ως «ελάχιστο λειτουργικό παράδειγμα» (minimal working example, MWE). Αυτό δεν σημαίνει ότι ο κώδικας λειτουργεί (αλλιώς δεν θα ρωτούσατε!), αλλά μάλλον σημαίνει ότι έχετε κάνει το καλύτερο δυνατό ώστε το παράδειγμα να είναι σαφές, αυτοτελές και ελάχιστο. Το τελευταίο σημαίνει ότι έχετε μόνο όσο περιεχόμενο χρειάζεται για να δείξετε το πρόβλημα.

### Πώς να δημιουργήσετε ένα ελάχιστο λειτουργικό παράδειγμα (MWE)

Πώς κατασκευάζετε ένα MWE; Συνήθως το πιο εύκολο είναι να ξεκινήσετε από

```latex
\documentclass{article}
\usepackage[T1]{fontenc}
\begin{document}
Text
\end{document}
```

και να προσθέσετε γραμμές μία προς μία μέχρι να φαίνεται το πρόβλημα. Μπορείτε να προσπαθήσετε να περικόψετε το πραγματικό αρχείο σας, αλλά αυτό ίσως να είναι πιο δύσκολο.

<p 
  class="hint">Αν χρειάζεστε περισσότερο κείμενο ώστε να δείξετε αλλαγές σελίδας και άλλα ζητήματα, τότε πακέτα όπως το <code>lipsum</code> μπορούν να χρησιμοποιηθούν για να παράξουν ασυνάρτητες παραγράφους κειμένου, διατηρώντας ταυτόχρονα το παράδειγμά σας σύντομο.</p>


### Αρχείο καταγραφής

Κάτι που είναι καλό να έχετε διαθέσιμο είναι το αρχείο καταγραφής σας. Αυτό δημιουργείται από το LaTeX κάθε φορά που το εκτελείτε και έχει το ίδιο όνομα με το κύριο αρχείο σας, αλλά κατάληξη `.log`.

<p 
  class="hint">Ανάλογα με το περιβάλλον του λειτουργικού συστήματός σας, ίσως χρειαστεί να «εμφανίσετε τις επεκτάσεις» των αρχείων για να καταλάβετε ποιο αρχείο είναι.</p>

Στο αρχείο καταγραφής μπορείτε πάντα να δείτε τα πλήρη μηνύματα σφάλματος. Τα μηνύματα σφάλματος του LaTeX προσπαθούν να είναι χρήσιμα, αλλά δεν είναι τα ίδια με τα μηνύματα σε επεξεργαστές κειμένου.

<p
  class="hint">Ορισμένοι επεξεργαστές LaTeX καθιστούν επίσης δύσκολη την προβολή του πλήρους κειμένου ενός σφάλματος, γεγονός που μπορεί να αποκρύψει σημαντικές λεπτομέρειες.</p>

Αν έχετε πρόβλημα, οι έμπειροι χρήστες του LaTeX θα ζητήσουν συχνά ένα αντίγραφο του αρχείου καταγραφής σας.
